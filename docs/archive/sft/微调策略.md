# 微调策略文档

## 项目概览

自动论文审稿系统,基于 LLaMA-3.1-8B-Instruct 微调,用于审阅研究论文

---

## 微调策略体系

### 一、上下文策略 (输入处理方式)

| 策略 | 描述 | 显存需求 | 适用场景 |
|------|------|----------|----------|
| **Full-context** | 完整论文输入 (~18,000 tokens) | H100 (80GB) | 全文推理,依赖长距离关系 |
| **Sliding-window** | 分段滑动窗口 (8192 tokens, 有重叠) | 4090 (24GB) ×5 | 显存受限,分段处理 |

#### 策略对比

**Full-context 策略:**
- 一次性输入完整论文内容
- 保持完整的论文上下文和逻辑关系
- 需要大显存支持 (H100)
- 适合需要全局理解的任务

**Sliding-window 策略:**
- 将长论文切分为多个重叠的窗口 (8192 tokens)
- 每个窗口独立处理,最后聚合结果
- 显存占用低,可在 4090 上训练
- 通过窗口重叠保持上下文连贯性
- 实际训练数据量更大 (每个样本产生多个片段)

---

### 二、训练方法 (优化策略)

#### 1. QLoRA (Quantized LoRA)

QLoRA 结合了量化和 LoRA (Low-Rank Adaptation) 技术,实现高效的参数微调。

**量化配置:**
```yaml
量化设置:
  - 4-bit 量化 (load_in_4bit: true)
  - 双重量化 (bnb_4bit_use_double_quant: true)
  - 计算精度: bfloat16
  - 梯度检查点启用

LoRA 参数:
  - Rank (r): 8
  - Alpha: 32
  - Dropout: 0.05
  - Bias: none
  - Task type: CAUSAL_LM
```

**训练超参数:**
```yaml
训练配置:
  - Batch size: 1 (per device)
  - 梯度累积步数: 4
  - 学习率: 2e-5
  - 训练轮数: 2 epochs
  - 学习率调度: cosine with warmup
  - Warmup 比例: 0.1
  - 混合精度: bf16
  - 评估策略: steps (每 100 步)
```

#### 2. DPO (Direct Preference Optimization)

在 QLoRA 基础上进一步对齐人类偏好,通过比较"好"和"坏"的输出来优化模型。

**两种拒绝样本策略:**

| 拒绝样本来源 | 描述 | 效果观察 |
|-------------|------|----------|
| **Base as rejected** | 原始 LLaMA-3.1 zero-shot 输出 | 损失接近0,学习过于简单 |
| **QLoRA as rejected** | QLoRA 微调后模型输出 | 损失非零,学习更有效 |

**DPO 超参数:**
```yaml
DPO 配置:
  - Beta (β): 0.1
  - Max length: 18000
  - 学习率: 2e-6
  - 训练轮数: 3 epochs
  - Batch size: 1 (per device)
  - 梯度累积步数: 4
  - 混合精度: bf16
  - 参考模型: None (DPO 不需要独立的参考模型)
```

**DPO 数据集格式:**
```json
{
  "prompt": "...论文内容和指令...",
  "chosen": "...GPT-4 聚合的高质量审稿...",
  "rejected": "...Base 或 QLoRA 模型生成的较差审稿..."
}
```

### 三、模型矩阵

```
┌─────────────────────────────────────────────────────────────┐
│                    微调策略组合                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Full Context                        Sliding Window        │
│  ┌─────────────────────┐           ┌─────────────────────┐ │
│  │ QLoRA              │           │ QLoRA              │ │
│  │ ↓                   │           │ ↓                   │ │
│  │ DPO (base rejected) │           │ DPO (base rejected) │ │
│  │ ↓                   │           │                     │ │
│  │ DPO (QLoRA rejected)│           │                     │ │
│  └─────────────────────┘           └─────────────────────┘ │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**实际训练的模型列表:**

1. `full_context_qlora`
   - 完整上下文 QLoRA 微调
   - 设备: H100
   - 训练轮数: 2 epochs
   - 上下文: ~18000 tokens

2. `full_context_dpo_base_as_rejected`
   - 基于 full_context_qlora 的 DPO 对齐
   - 拒绝样本: 原始 LLaMA-3.1 输出
   - 用于测试弱拒绝基线的改进效果

3. `full_context_dpo_qlora_as_rejected`
   - 基于 full_context_qlora 的 DPO 对齐
   - 拒绝样本: QLoRA 微调后模型输出
   - 更强的拒绝样本,评估是否能改进偏好学习

4. `sliding_window_qlora`
   - 滑动窗口 QLoRA 微调
   - 设备: 5× 4090 GPU
   - 窗口大小: 8192 tokens
   - 训练轮数: 2 epochs

5. `sliding_window_dpo_base_as_rejected`
   - 基于 sliding_window_qlora 的 DPO 对齐
   - 拒绝样本: 原始 LLaMA-3.1 输出

### 四、硬件配置

| 训练类型 | 设备 | 平台 | 显存利用 | 特点 |
|---------|------|------|----------|------|
| Full-context QLoRA | 1× H100 (80GB) | RunPod | 充分利用大显存 | 单卡训练,稳定高效 |
| Sliding-window QLoRA | 5× 4090 (24GB) | AutoDL | 分布式训练,每卡低负载 | 多卡并行,资源灵活 |
| DPO | 基于以上 | 基于以上 | 在 QLoRA 基础上继续训练 | 需要加载 QLoRA adapter |

**硬件选择策略:**
- **H100**: 处理超长上下文的首选,避免多卡通信开销
- **4090 ×5**: 多卡并行,但大上下文前向传播仍受单卡限制
- **4-bit 量化**: 显著降低显存占用,使得在有限显存上训练成为可能

### 五、评估维度

#### 评分预测指标 (Regression)

| 指标 | 公式 | 意义 |
|------|------|------|
| **MAE** | $\frac{1}{n}\sum_{i=1}^{n}|y_i - \hat{y}_i|$ | 平均绝对误差,衡量预测偏离度 |
| **RMSE** | $\sqrt{\frac{1}{n}\sum_{i=1}^{n}(y_i - \hat{y}_i)^2}$ | 均方根误差,对大误差更敏感 |
| **Pearson** | $\frac{\sum(x-\bar{x})(y-\bar{y})}{\sqrt{\sum(x-\bar{x})^2\sum(y-\bar{y})^2}}$ | 皮尔逊相关系数,衡量趋势一致性 |

**指标解读:**
- **MAE + RMSE 组合使用**: MAE 反映平均误差,RMSE 强调大误差
- **Low MAE, Low RMSE**: 预测准确且稳定
- **Low MAE, High RMSE**: 总体良好但存在一些严重错误
- **Pearson**: 衡量预测和真实评分的线性关系,不直接反映误差大小

#### 文本生成指标 (NLG)

| 指标 | 描述 | 特点 |
|------|------|------|
| **BLEU-4** | 衡量 n-gram 重叠度 | 偏向精确率 (precision),对输出过短有惩罚 |
| **ROUGE-1** | Unigram 重叠 | 词汇级别的召回率 |
| **ROUGE-2** | Bigram 重叠 | 短语级别的召回率 |
| **ROUGE-L** | 最长公共子序列 | 保持顺序的一致性 |

**指标特点:**
- BLEU: 更注重精确率,对同义改写不友好
- ROUGE: 更注重召回率,衡量信息覆盖度
- 这些是表面相似度指标,不一定完全等同于真实质量

### 六、实验结果与发现

#### 评估结果对比

| Model | OQ_MAE↓ | OQ_RMSE↓ | OQ_Pearson↑ | Conf_MAE↓ | Conf_RMSE↓ | Conf_Pearson↑ | BLEU-4↑ | ROUGE-1↑ | ROUGE-2↑ | ROUGE-L↑ |
|-------|---------|----------|-------------|-----------|------------|---------------|---------|----------|----------|----------|
| llama3___1 (base) | 1.9633 | 2.0615 | -0.1693 | 1.1567 | 1.9174 | -0.1332 | 4.28 | 27.46 | 8.27 | 14.91 |
| full_context_qlora | 0.809 | 1.0429 | -0.1123 | 0.5426 | 0.694 | 0.0063 | 6.33 | 27.96 | 10.38 | 15.28 |
| full_context_dpo_qlora_as_rejected | 1.18 | 1.4398 | -0.1375 | 0.675 | 0.8228 | -0.0892 | 5.77 | 29.52 | 10.66 | 14.32 |
| sliding_window_dpo_base_as_rejected | 0.7341 | 0.9386 | 0.0965 | 0.5388 | 0.6963 | 0.075 | 6.77 | 28.28 | 10.96 | 15.88 |

#### 关键发现

1. **DPO 确实提升生成质量**
   - 相比纯 QLoRA,DPO 进一步改善了生成质量
   - 在文本生成指标 (BLEU, ROUGE) 上有提升

2. **QLoRA-based 拒绝样本更有效**
   - 使用 Base 作为拒绝样本时,模型损失接近 0,学习过于简单
   - 使用 QLoRA 作为拒绝样本时,产生非零损失曲线,学习更有效
   - 说明拒绝样本的质量和难度对 DPO 训练至关重要

3. **Sliding window 模型泛化能力惊人**
   - 尽管只有 8192 tokens 上下文 (vs 18000),性能表现优异
   - 在多个指标上超越了 full-context 模型
   - 可能的原因:
     - 数据增强效应: 每个样本被分成多个片段,实际训练数据量更大
     - 避免了长序列训练中的梯度消失问题
     - 更精细的局部特征学习

4. **最佳模型**
   - **sliding_window_dpo_base_as_rejected** 综合表现最佳
   - 在评分预测 (MAE, RMSE) 和文本生成 (BLEU, ROUGE) 上都有优势

5. **Full-context DPO 的困惑**
   - `full_context_dpo_qlora_as_rejected` 的表现不如预期
   - 可能原因:
     - DPO 在超长上下文下的训练不稳定
     - 拒绝样本 (QLoRA) 可能已经足够好,导致对比信号不够强
     - 需要调整 DPO 的超参数 (如 beta 值)

### 七、训练流程

```
数据准备阶段
├── OpenReview 数据爬取 (ICML, NIPS, ICLR, CVPR)
├── Nougat-OCR 解析 PDF
├── GPT-4-mini 格式化审稿内容
└── 生成结构化训练数据
       ↓
QLoRA 微调 (SFT)
├── Full-context (H100) 或 Sliding-window (4090×5)
├── 4-bit 量化 + LoRA (r=8)
└── 2 epochs, lr=2e-5
       ↓
生成拒绝样本 (用于 DPO)
├── Base model (zero-shot) 生成
├── QLoRA model 生成
└── 构建 DPO 数据集
       ↓
DPO 对齐训练
├── 加载 QLoRA adapter
├── 使用 chosen (GPT-4) 和 rejected (Base/QLoRA)
├── 3 epochs, lr=2e-6, beta=0.1
└── 输出对齐后的模型
       ↓
评估
├── 评分预测 (MAE, RMSE, Pearson)
├── 文本生成 (BLEU-4, ROUGE-1/2/L)
└── 多模型对比分析
```

### 八、技术亮点

1. **对比实验设计**
   - 系统对比了不同上下文长度的影响

   - 结合评分预测和文本生成两套指标
   - 既关注准确性也关注质量
   - 多维度评估模型性能

4. **实用的训练 pipeline**
   - 从数据准备到模型评估的完整流程
   - 代码结构清晰,易于复现和扩展
   - 使用 LLaMA-Factory 快速原型,自定义脚本深度优化

## 总结

本项目通过系统的实验设计,探索了多种微调策略在自动审稿任务中的应用。研究发现:

- **QLoRA** 提供了高效的参数微调方案
- **DPO** 能进一步提升模型生成质量,但拒绝样本的选择至关重要
- **Sliding window** 策略在资源受限环境下展现出优异性能
- **综合指标** 表明微调后的模型显著超越了原始 LLaMA-3.1 基线

最佳模型 `sliding_window_dpo_base_as_rejected` 在评分准确性和文本质量上都有显著提升,为自动审稿系统的实际应用奠定了基础。
