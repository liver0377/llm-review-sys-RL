# 数据爬取与处理流程详解

## 概述

本文档详细说明了从OpenReview爬取数据到构建最终训练数据集的完整流程。数据流程包括论文爬取、PDF下载、文本解析、评审聚合、数据集构建和DPO数据准备等多个阶段。

---

## 一、整体数据流程图

```
┌─────────────────────────────────────────────────────────────────┐
│                    数据处理Pipeline                            │
└─────────────────────────────────────────────────────────────────┘

1. OpenReview爬取
   ├── 论元数据 (标题、作者、PDF链接)
   └── 真实评审内容 (Official_Review, Official_Comment, Meta_Review)
       ├── review_text: 人类评审文本
       ├── rating: 评分 (1-10)
       └── confidence: 置信度 (1-5)

2. PDF下载
   └── 从OpenReview下载论文PDF文件

3. PDF解析
   └── 使用Nougat-OCR解析PDF为Markdown格式

4. 论文清洗
   ├── 移除参考文献
   ├── 移除致谢部分
   ├── 移除脚注
   └── 清理LaTeX残留

5. 评审聚合
   ├── 输入: 多条真实人工评审
   ├── 使用: GPT-4o-mini
   └── 输出: 结构化评审
       ├── ### Key Points
       ├── ### Strengths and Weaknesses
       └── ### Suggestions for Improvement

6. 草稿数据集构建
   └── 合并论文内容和聚合评审

7. DPO数据集构建
   ├── chosen: GPT-4聚合的真实评审
   └── rejected: 模型生成的评审
```

## 二、OpenReview数据爬取

### 功能描述

从OpenReview平台爬取顶级会议的论文及其评审数据。

### 核心代码

**文件**: `data/data_crawler.py`

**爬取流程**:
```python
import openreview

# 初始化OpenReview客户端
client = openreview.api.OpenReviewClient(
    baseurl="https://api2.openreview.net",
    username=username,
    password=password
)

# 获取会议的所有子群组
subgroups = get_all_subgroups(client, conference)

# 爬取每个venue的论文和评审
for venue_id in subgroups:
    invitation_format = f"{venue_id}/-/{submission_name}"
    submissions = client.get_all_notes(invitation=invitation_format, details="replies")

    for paper in submissions:
        # 提取论文元数据
        paper_info = {
            "title": paper.content.get("title"),
            "conference": conference_name,
            "pdf_url": f"https://openreview.net/pdf?id={paper.id}",
            "year": extract_year(paper),
            "reviews": []
        }

        # 提取评审信息
        allowed_review_types = ["Official_Review", "Official_Comment", "Meta_Review"]
        for reply in paper.details["replies"]:
            if invitation_type in allowed_review_types:
                review_data = {
                    "type": invitation_type,
                    "content": reply.get("content", {}),
                    "ratings": {}
                }

                # 提取评分和置信度
                if "rating" in reply.get("content", {}):
                    review_data["ratings"]["rating"] = reply["content"]["rating"]
                if "confidence" in reply.get("content", {}):
                    review_data["ratings"]["confidence"] = reply["content"]["confidence"]

                paper_info["reviews"].append(review_data)
```

### 数据结构

**输出格式** (`results.json`):

- **年份过滤**: 只爬取2020年及以后的论文

- **增量爬取**: 跳过已存在的会议文件夹
- **错误处理**: 自动处理API异常和格式错误

## 四、PDF解析

### 功能描述

使用Nougat-OCR工具将PDF文件解析为Markdown格式的文本。

### 核心代码

**文件**: `data/pdf_parser.py`

**解析流程**:
```python
def process_single_pdf(args):
    pdf_path, output_dir = args

    # 使用Nougat解析PDF
    command = ["nougat", pdf_path, "-o", output_dir, "-m", "0.1.0-base"]

    try:
        result = subprocess.run(command)
        base_name = os.path.splitext(os.path.basename(pdf_path))[0]
        output_file = os.path.join(output_dir, f"{base_name}.mmd")

        return True
    except Exception as e:
        print(f"解析失败: {pdf_path} - {e}")
        return False
```

- **输出格式**: Markdown (.mmd文件)

## 五、论文清洗

### 功能描述

清理解析后的论文文本，移除不必要的部分，提高数据质量。

### 核心代码

**文件**: `data/paper_cleaner.py`

**清洗流程**:
```python
def extract_key_info_from_paper(content, max_tokens=30000):
    """提取论文关键信息，移除不需要的部分"""

    # 提取标题
    title_match = re.search(r'^#\s+(.*?)$', content, re.MULTILINE)
    title = title_match.group(1) if title_match else "Unknown Title"

    # 清理内容
    cleaned_content = content

    # 移除参考文献
    cleaned_content = re.sub(
        r'(?:^|\n)#{1,3}\s+(?:References|Bibliography|参考文献)[\s\S]*$',
        '', cleaned_content, flags=re.IGNORECASE
    )

    # 移除致谢部分
    cleaned_content = re.sub(
        r'(?:^|\n)#{1,3}\s+(?:Acknowledgements?|Acknowledgments?)[\s\S]*$',
        '', cleaned_content, flags=re.IGNORECASE
    )

    # 移除脚注
    cleaned_content = re.sub(r'Footnote \d+:.*?\n', '', cleaned_content)

    # 移除引用标记 [1], [2,3], etc.
    cleaned_content = re.sub(r'\[\d+(?:,\s*\d+)*\]', '', cleaned_content)

    # 移除LaTeX残留标记
    cleaned_content = re.sub(r'\\[a-zA-Z]+(\{[^}]*\})*', '', cleaned_content)

    # 移除中文内容（如果有）
    cleaned_content = re.sub(r'[\u4e00-\u9fff]+', '', cleaned_content)

    # 组合最终结果
    final_content = f"# {title}\n\n{cleaned_content.strip()}"

    return final_content
```

### 清理规则

| 清理项 | 目的 | 正则表达式 |
|--------|------|-----------|
| **参考文献** | 减少无关内容 | `References\|Bibliography\|参考文献` |
| **致谢** | 移除非核心内容 | `Acknowledgements?\|Acknowledgments?` |
| **脚注** | 简化文本 | `Footnote \d+:` |
| **引用标记** | 避免数字干扰 | `\[\d+(?:,\s*\d+)*\]` |
| **LaTeX标记** | 移除格式残留 | `\\[a-zA-Z]+(\{[^}]*\})*` |
| **中文内容** | 统一语言 | `[\u4e00-\u9fff]+` |

## 六、评审聚合

### 功能描述

使用GPT-4o-mini将多条真实人工评审聚合成一条结构化的评审。

### 核心代码

**文件**: `data/review_summarizer.py`

**聚合流程**:
```python
from openai import OpenAI

class ReviewAggregator:
    def __init__(self, api_key_name):
        api_key = os.getenv(api_key_name)
        self.client = OpenAI(api_key=api_key)

    def aggregate_reviews(self, reviews):
        """使用GPT-4o-mini聚合多条评审"""

        # 估算token数量
        total_chars = sum(len(review) for review in reviews)
        estimated_tokens = total_chars // 4

        # 如果评论太长，分批处理
        if estimated_tokens > 6000:
            batches = self._split_into_batches(reviews)
            summaries = [self._process_single_batch(batch) for batch in batches]
            return self._merge_summaries(summaries)
        else:
            return self._process_single_batch(reviews)

    def _process_single_batch(self, reviews):
        """处理单批次评审"""
        numbered_reviews = "\n\n".join([
            f"Review {i+1}: {review}"
            for i, review in enumerate(reviews)
        ])

        prompt = f"""
        Please generate a concise summary based on following review(s) with following three sections:
        1. Key points
        2. Strengths and weaknesses
        3. Suggestions for improvement

        Critical Requirements:
        - Strictly maintain exact core viewpoints of original review(s)
        - Absolute prohibition on adding any new information
        - Summarize ONLY using provided review content
        - Preserve verbatim any specific technical suggestions or terms
        - Ensure summary captures full depth and nuance of original reviews
        - Match original review's academic tone and technical precision
        - Reduce length by 20-30% without losing critical information
        - When making suggestions, write as if reviewers are directly addressing authors

        Format Requirements:
        - Use "### Key Points", "### Strengths and Weaknesses", and "### Suggestions for Improvement"
        - In Strengths and Weaknesses section, write "Strengths:" and "Weaknesses:" on separate lines

        Review(s):
        {numbered_reviews}
        """

        response = self.client.chat.completions.create(
            model="gpt-4o-mini",
            messages=[{"role": "user", "content": prompt}],
            temperature=0.2
        )

        return response.choices[0].message.content
```

### 关键Prompt设计

**严格约束**:
- ✅ 保持原始评审的核心观点
- ❌ 绝对禁止添加新信息
- ✅ 只基于提供的评审内容进行总结
- ✅ 保持学术语调和技术精确性
- ✅ 字面保留具体的技术建议和术语

**输出格式**:
```markdown
### Key Points
This paper presents...

### Strengths and Weaknesses
Strengths:
- The model exhibits...

Weaknesses:
- The reliance on assumptions...

### Suggestions for Improvement
We recommend that the authors improve...
```

### 分批处理机制

当评审总长度超过6000 tokens时：
1. 将评审分成多个批次 (每批约6000 tokens)
2. 分别处理每个批次
3. 合并所有批次的摘要
4. 确保最终摘要连贯完整

## 七、草稿数据集构建

### 功能描述

将解析后的论文内容与聚合的评审数据合并，构建完整的训练数据集。

### 核心代码

**文件**: `data/prepare_draft_dataset.py` (或 `create_draft_dataset.py`)

**构建流程**:
```python
def create_paper_review_dataset(aggregated_reviews, parsed_pdfs, output_path):
    """创建论文评审数据集"""

    for review in aggregated_reviews:
        paper_id = review['id']

        if paper_id in parsed_pdfs:
            pdf_data = parsed_pdfs[paper_id]

            # 提取论文内容
            paper_content = ""
            if 'abstract' in pdf_data:
                paper_content += f"{pdf_data['abstract']}\n\n"
            if 'sections' in pdf_data:
                for section in pdf_data['sections']:
                    if 'heading' in section and 'text' in section:
                        paper_content += f"{section['heading']}\n{section['text']}\n\n"

            review_content = review.get('review') or review.get('aggregated_review', '')

            # 计算平均评分和置信度
            valid_ratings = [r for r in review.get('original_ratings', []) if r != -1]
            valid_confidences = [c for c in review.get('original_confidences', []) if c != -1]

            if valid_ratings and valid_confidences:
                avg_rating = sum(valid_ratings) / len(valid_ratings)
                avg_confidence = sum(valid_confidences) / len(valid_confidences)

                data_item = {
                    "id": paper_id,
                    "title": review['title'],
                    "conference": review.get('conference', ''),
                    "year": review.get('year', ''),
                    "paper_content": paper_content,
                    "aggregated_review": review_content,
                    "original_ratings": review['original_ratings'],
                    "original_confidences": review['original_confidences'],
                    "avg_rating": round(avg_rating, 2),
                    "avg_confidence": round(avg_confidence, 2)
                }

                new_data.append(data_item)

    # 保存数据集
    combined_data = existing_data + new_data
    with open(output_path, 'w', encoding='utf-8') as f:
        json.dump(combined_data, f, ensure_ascii=False, indent=2)
```

### 数据匹配逻辑

1. **按paper_id匹配**: 从聚合评审和解析的PDF数据中匹配同一篇论文
2. **验证数据完整性**: 确保有效的评分和置信度
3. **计算统计量**: 计算平均评分和置信度
4. **增量更新**: 合并已有数据和新数据

## 八、DPO数据集构建

### 功能描述

基于模型推理结果，构建用于DPO训练的chosen/rejected对数据集。

### 核心代码

**文件**: `data/create_dpo_dataset.py`

**构建流程**:
```python
import json

# 加载推理结果
input_path = "rejected_dataset/inference_base_model.jsonl"
output_path = "llama_factory_dataset/dpo_pair_llama3.json"

dpo_data = []

with open(input_path, "r", encoding="utf-8") as f:
    for line in f:
        item = json.loads(line)

        # 构建prompt
        prompt = f"{item['instruction'].strip()}\n\n{item['input'].strip()}"

        # chosen: GPT-4聚合的真实评审
        chosen = item["output"].strip()

        # rejected: 模型生成的评审
        rejected = item["generated_output"].strip()

        dpo_data.append({
            "prompt": prompt,
            "chosen": chosen,
            "rejected": rejected
        })

# 保存DPO数据集
with open(output_path, "w", encoding="utf-8") as f:
    json.dump(dpo_data, f, ensure_ascii=False, indent=2)
```

### 数据来源说明

| DPO字段 | 来源 | 数据性质 | 说明 |
|---------|------|----------|------|
| **prompt** | 论文内容+评审指令 | 中性 | 不偏向任何一方 |
| **chosen** | GPT-4聚合的真实评审 | ✅ 真实数据 | 基于OpenReview人工评审 |
| **rejected** | 模型inference输出 | ❌ 生成数据 | Base/QLoRA模型生成 |

## 十三、常见问题与解决方案

### 1. OpenReview API限流

**问题**: 请求过于频繁，返回HTTP 429

**解决方案**:
- 增加请求延迟 (3-5秒)
- 使用指数退避策略
- 分批次处理

### 2. PDF下载失败

**问题**: 某些PDF无法下载

**解决方案**:
- 检查URL有效性
- 添加重试机制 (最多5次)
- 跳过失败的PDF，记录日志

### 3. Nougat解析失败

**问题**: 某些PDF解析不成功

**解决方案**:
- 检查PDF格式是否标准
- 尝试不同的Nougat模型版本
- 记录失败的PDF，单独处理

### 4. GPT-4 API调用失败

**问题**: API返回错误或超时

**解决方案**:
- 实现重试机制
- 分批处理长评审
