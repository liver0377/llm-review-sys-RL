# 模型评估指南

本指南说明如何评估模型性能，包括 Base Model 和 SFT Model 的对比。

## 📊 评估系统架构

```
测试数据 (sft_test.json)
    ↓
Inference (生成预测)
    ↓
评估脚本 (计算指标)
    ↓
结果文件 (summary.csv)
```

## 🚀 快速开始

### ⚡ vLLM 快速评估（推荐）

#### 🚀 标准评估流程（单 GPU）

**Step 1: 创建评测子集（可选）**
```bash
python scripts/create_eval_subset.py \
  --input data/openreview_dataset/sft_test.json \
  --num_samples 50
```
生成文件：`data/openreview_dataset/sft_test_subset_50.json`

**Step 2: 启动 vLLM 服务**
```bash
# 前台运行（可查看实时日志）
python serve/vllm_review_service.py

# 或后台运行
nohup python serve/vllm_review_service.py > vllm_service.log 2>&1 &
```

等待服务启动完成（约 1-2 分钟），看到以下日志表示成功：
```
✅ vLLM 引擎初始化完成
   Tensor Parallel: 1
```

**Step 3: 运行批量推理**
```bash
python scripts/fast_inference_vllm.py \
  --data "data/openreview_dataset/sft_test_subset_50.json" \
  --output "eval/results_vllm.jsonl" \
  --batch_size 10
```

**Step 4: 评估结果**
```bash
python eval/eval.py --model_name "results_vllm"
cat eval/eval_results/summary.csv
```

#### 🔥 多GPU加速推理

**环境变量说明：**

| 环境变量 | 默认值 | 说明 |
|---------|--------|------|
| `TENSOR_PARALLEL` | 1 | GPU 数量（1, 2, 4, 8） |
| `GPU_MEMORY_UTILIZATION` | 0.9 | 显存利用率（0-1） |
| `MAX_MODEL_LEN` | 32768 | 最大上下文长度 |

**多GPU启动命令：**

```bash
# 使用 2 个 GPU
TENSOR_PARALLEL=2 python serve/vllm_review_service.py

# 使用 4 个 GPU
TENSOR_PARALLEL=4 GPU_MEMORY_UTILIZATION=0.85 python serve/vllm_review_service.py

# 使用 8 个 GPU（最大吞吐）
TENSOR_PARALLEL=8 python serve/vllm_review_service.py

# 多GPU + 切换模型
TENSOR_PARALLEL=4 VLLM_MODEL_NAME=dpo_base_qwen3_8b python serve/vllm_review_service.py
```

**性能对比（531 样本）：**

| GPU 配置 | 推理时间 | 吞吐量 | 显存/GPU |
|---------|---------|--------|---------|
| 1×A100 | ~40 分钟 | 13 样本/分钟 | 18GB |
| 2×A100 | ~22 分钟 | 24 样本/分钟 | 9GB |
| 4×A100 | ~12 分钟 | 44 样本/分钟 | 5GB |
| 8×A100 | ~7 分钟 | 76 样本/分钟 | 3GB |

**选择建议：**
- 小数据集（<100）：1-2 GPU
- 标准评估（500）：2-4 GPU
- 大规模（1000+）：4-8 GPU

## 📝 详细说明

### 1. 生成模型预测

**基础用法：**
```bash
python train/code/inference.py --model qwen3_8b_base
```

**完整参数：**
```bash
python train/code/inference.py \
  --model qwen3_8b_base \
  --eval_data data/openreview_dataset/sft_test.json \
  --max_input_tokens 15000 \
  --max_output_tokens 2000
```

**参数说明：**
- `--model`: 模型名称（见下方支持的模型列表）
- `--eval_data`: 测试数据路径（默认：`data/openreview_dataset/sft_test.json`）
- `--max_input_tokens`: 最大输入 token 数（默认：15000）
- `--max_output_tokens`: 最大输出 token 数（默认：2000）
- `--checkpoint`: 指定 checkpoint（如 `checkpoint-800`）

### 2. 评估模型

**基础用法：**
```bash
python eval/eval.py --model_name qwen3_8b_base
```

**完整参数：**
```bash
python eval/eval.py \
  --model_name qwen3_8b_base \
  --save_dir eval_results
```

**参数说明：**
- `--model_name`: 模型名称（需与 inference 时一致）
- `--save_dir`: 结果保存目录（默认：`eval_results`）

### 3. 查看结果

**查看对比表格：**
```bash
cat eval/eval_results/summary.csv
```

**查看单个模型详细结果：**
```bash
cat eval/eval_results/qwen3_8b_qlora_ratings_eval.json
```

## 📈 评估指标

### Rating 预测指标

- **MAE** ↓: 平均绝对误差（越小越好）
- **RMSE** ↓: 均方根误差（越小越好）
- **Pearson** ↑: 相关系数（越大越好）

评估对象：
- **Overall Quality**: 整体质量评分（1-10 分）
- **Review Confidence**: 评审置信度（1-5 分）

### NLG 指标

- **BLEU-4** ↑: 4-gram 精确率
- **ROUGE-1** ↑: Unigram 召回率
- **ROUGE-2** ↑: Bigram 召回率
- **ROUGE-L** ↑: 最长公共子序列

## 📁 输出文件

### Inference 输出

- **位置**: `eval/results_{model_name}.jsonl`
- **格式**: JSONL（每行一个 JSON 对象）
- **内容**: 包含输入、真实输出、模型预测

### Eval 输出

- **详细结果**: `eval/eval_results/{model_name}_ratings_eval.json`
  - 包含所有指标的详细数值
- **汇总表格**: `eval/eval_results/summary.csv`
  - 包含所有已评估模型的对比

## ⚠️ 注意事项

1. **内存需求**: 推理需要约 16-20GB 显存（bfloat16）
2. **推理时间**: 每个模型约 1-2 小时（531 个测试样本）
3. **断点续传**: inference 脚本支持断点续传，可以随时中断
4. **模型名称**: 确保 `--model` 和 `--model_name` 参数一致

## 🔧 常见问题

### Q1: 如何加快评估速度？

减少 `max_input_tokens`（但可能影响质量）：
```bash
python train/code/inference.py \
  --model qwen3_8b_base \
  --max_input_tokens 8000
```

### Q2: 内存不足怎么办？

使用更小的 batch size 或梯度检查点，或者使用更少的样本。

### Q3: 如何只评估部分数据？

修改 `train/code/inference.py` 中的数据加载逻辑，或创建一个小的测试集子集。

### Q4: 评估结果在哪里？

所有结果保存在 `eval/` 目录：
- 预测结果：`eval/results_{model_name}.jsonl`
- 评估结果：`eval/eval_results/`
- 汇总表格：`eval/eval_results/summary.csv`
